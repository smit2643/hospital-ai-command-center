# OCR Review and Signature Flow

## Objective
Ensure extracted OCR data is reviewed and corrected before final save, and optionally sent for patient signature from the same screen.

## Flow
1. Upload document from `/documents/upload/`
2. Trigger OCR from patient document table
3. Open OCR review page (`/documents/<id>/ocr/result/`)
4. Edit extracted fields:
- patient_name
- report_date
- hospital_name
- doctor_name
- tests_json
- notes
5. (Optional) enable `send_for_signature` and provide signer email
6. Save reviewed data
7. Signature request status appears as `SENT/VIEWED/SIGNED`

## Signed Visibility
Document table now shows signature state:
- `NOT SENT`
- `SENT`
- `VIEWED`
- `SIGNED`

## Sample Test Asset
- `sample_data/hospital_dummy_lab_report.png`
- Generated by `scripts/generate_dummy_lab_report.py`

## Notes on OCR Provider
- Default provider: open-source Tesseract (`OCR_PROVIDER=tesseract`)
- Optional Gemini provider can be enabled via env vars when API key is available.
